# Release 0.1.0-demo.1

Stage: `demo`

Tag target: `v0.1.0-demo.1`

Release branch: `release/0.1.0-demo.1`

Date: `2026-03-27`

## Purpose

This release is the first demo baseline for OpenJobEngine.

It is intended to show the product direction end to end:

- multi-source job ingestion
- normalization and enrichment
- job history and source observations
- candidate profiles
- resume PDF parsing
- explainable matching
- operational metrics

## Included capabilities

### Aggregation

- `Computrabajo` provider
- `Adzuna` provider
- `Greenhouse` provider

### Data pipeline

- normalization to a canonical job model
- enrichment of salary, work mode, seniority, location, skills and languages
- deduplication by canonical key
- persistence in SQLite by default, PostgreSQL optional
- append-only job history plus source observation tracking

### Candidate-facing backend

- profile CRUD by API
- saved searches
- profile alerts
- resume import from PDF
- heuristic extraction with warnings and confidences
- deterministic explainable matching

### Operations

- worker process for scheduled collection
- Swagger enabled
- provider quality metrics
- build validation in CI for Git Flow branches
- semantic versioning baseline exposed in Swagger and `X-OpenJobEngine-Version`

## Demo flow

1. Start the API with SQLite.
2. Enable one provider in appsettings or environment variables.
3. Run a collection.
4. Create a candidate profile.
5. Upload a sample resume PDF.
6. Request matches for the profile.
7. Show job history for one aggregated posting.
8. Show metrics and Swagger version.

Reference requests:

- `docs/http/OpenJobEngine.http`

## Known limitations

- no auth or API keys yet
- no health checks yet
- schema still uses `EnsureCreated`, not EF migrations
- all providers are disabled by default until configured
- resume parsing is heuristic, not LLM-based
- worker scheduling is still basic and single-interval
